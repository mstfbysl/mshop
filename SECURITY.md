## Reporting a Vulnerability

If you discover a security vulnerability, please **do not** open a public issue. Instead, please report it via one of the following methods:

### Preferred Method

1. Open a security advisory on the GitHub repository (if available)
2. Or email the maintainer directly at the contact information available on the live site

### Information to Include

When reporting a vulnerability, please include:

- **Type of vulnerability** (e.g., XSS, CSRF, SQL injection, etc.)
- **Location** of the affected code (file, line number, or URL)
- **Steps to reproduce** the vulnerability
- **Potential impact** of the vulnerability
- **Suggested fix** (if you have one)

### Response Time

We aim to:

- Acknowledge receipt of your report within 48 hours
- Provide an initial assessment within 7 days
- Keep you informed of our progress every 7-14 days

### Disclosure Policy

- We will work with you to understand and validate the issue
- Once validated, we will work on a fix
- We will coordinate disclosure with you after the fix is available
- Credit will be given in the security advisory if desired

## Security Best Practices

When using this project:

- Keep dependencies up to date
- Review and understand any environment variables or configuration
- Follow security best practices when deploying
- Regularly audit your deployment

## Thank You

We appreciate your help in keeping MonoShop and its users safe!
